# Jenkinsfile - Quick Reference

## 📋 What This Jenkinsfile Does

```
START
  ↓
1️⃣ Initialize (create result directories)
  ↓
2️⃣ Generate Test Cases (using jira-test-generator)
  ↓
3️⃣ Verify Test Cases (check if file exists)
  ↓
4️⃣ Execute Tests (using manual-test-runner)
  ↓
5️⃣ Collect Results (organize output files)
  ↓
6️⃣ Parse Metrics (extract statistics)
  ↓
7️⃣ Generate Report (create HTML report)
  ↓
8️⃣ Archive Results (save all artifacts)
  ↓
END
```

---

## 🎯 Input Parameters

| Parameter | Type | Default | Purpose |
|-----------|------|---------|---------|
| JIRA_ISSUE | String | SCRUM-2 | Jira issue to test |
| APP_URL | String | http://localhost:3000 | Application URL |
| SKIP_TEST_GENERATION | Boolean | false | Skip generation step |

---

## 📁 Output Structure

```
results/
├── SCRUM-2-test-cases.md       ← Generated test cases
├── SCRUM-2-test-results.md     ← Test execution results ⭐ KEY FILE
├── METRICS.txt                  ← Pass/fail statistics
├── REPORT.html                  ← Visual HTML report
├── BUILD_STATUS.txt             ← Build success/failure
├── evidence/                    ← Screenshots of failures
│   ├── screenshot-TC-001.png
│   ├── screenshot-TC-004.png
│   └── ...
└── logs/
    ├── generation.log           ← Test generation logs
    └── execution.log            ← Test execution logs
```

## ⚙️ Installation (2 minutes)

### Local Jenkins Setup

```bash
# 1. Ensure Claude CLI is installed
npm install -g @anthropic-ai/claude-code

# 2. Verify it works
claude-code --version

# 3. Authenticate with Atlassian (one-time)
claude-code login
```

### Create Jenkins Job

1. Open http://localhost:8080
2. Click **New Item** → **Pipeline**
3. Name it: `NeuralQA-Test-Automation`
4. Under Pipeline:
   - Definition: `Pipeline script from SCM`
   - SCM: `Git`
   - Repository URL: `/path/to/repo`
   - Script Path: `Jenkinsfile`
5. Click **Save**

---

## 🚀 Run the Pipeline

### Method 1: GUI (Easiest)

1. Go to job page
2. Click **Build with Parameters**
3. Set JIRA_ISSUE: `SCRUM-2`
4. Click **Build**

### Method 2: Command Line

```bash
# Using curl
curl -X POST http://localhost:8080/job/NeuralQA-Test-Automation/buildWithParameters \
  -F JIRA_ISSUE=SCRUM-3 \
  -F APP_URL=http://localhost:3000
```

### Method 3: Automated (Scheduled)

In Jenkins job configuration:
```
Build Triggers:
- Build periodically
- Schedule: H 9 * * * (daily at 9 AM)
```

## 🎓 Common Tasks

### Generate & Execute Tests for SCRUM-3

1. Click **Build with Parameters**
2. Change JIRA_ISSUE to: `SCRUM-3`
3. Click **Build**
4. Wait for completion
5. Results in: `workspace/results/`

### Re-execute Existing Tests (Skip Generation)

1. Click **Build with Parameters**
2. Set SKIP_TEST_GENERATION: ✅
3. Click **Build**
4. Only executes manual-test-runner

```
Jenkins Home → NeuralQA-Test-Automation → Build History
```

Each build shows:
- Build number
- Timestamp
- Status (✅ or ❌)
- Execution time

### Download Artifacts

```
Jenkins Home → NeuralQA-Test-Automation → Build #X → Artifacts

Download:
- results.md files
- METRICS.txt
- Evidence screenshots
```

## 🐛 Troubleshooting Quick Fix

| Problem | Solution |
|---------|----------|
| "claude-code not found" | `npm install -g @anthropic-ai/claude-code` |
| "Claude not authenticated" | Run `claude-code login` |
| "Test cases not generated" | Check `results/logs/generation.log` |
| "Tests didn't execute" | Check `results/logs/execution.log` |
| "No results folder" | Ensure Jenkins has write permissions |
| "Permission denied" | Run Jenkins with proper user permissions |

## 💡 Tips & Tricks

### Quickly View Latest Results
```bash
# Navigate to latest results
cd $(ls -td /var/lib/jenkins/workspace/NeuralQA-Test-Automation/results | head -1)
cat METRICS.txt
```

### Monitor Multiple Issues
```bash
# Build multiple issues sequentially
for issue in SCRUM-2 SCRUM-3 SCRUM-4; do
  curl -X POST http://localhost:8080/job/NeuralQA-Test-Automation/buildWithParameters \
    -F JIRA_ISSUE=$issue
done
```

### Parse Results Programmatically
```bash
# Extract pass rate
grep "Pass Rate:" results/METRICS.txt
```
